📦 Модуль оптимизации упаковки коробок (Box Packer)

Python-модуль для решения задачи упаковки коробок в контейнеры (**Bin Packing / Knapsack**).  
Позволяет проверить возможность размещения заказа, оптимизировать количество изделий под целевой коэффициент заполнения и учесть физические ограничения укладки.

## 🧠 Алгоритм упаковки
В соответствии с п. 3.3 ТЗ реализован подход **First-Fit Decreasing (FFD)** с постобработкой ограничений:

1. **Сортировка**: все коробки заказа сортируются по убыванию объёма (`O(N log N)`).
2. **Последовательное заполнение**: контейнеры обрабатываются в порядке приоритета (`V1 → V2 → V3`).
3. **Послойная упаковка**:
   - Коробка размещается в текущем слое, если найдено свободное место.
   - Кандидаты позиций генерируются по правым/верхним граням уже уложенных коробок + `(0,0)`.
   - При нехватке места создаётся новый слой (если позволяет высота контейнера).
4. **Валидация**: на каждом шаге проверяются:
   - Границы контейнера и отсутствие пересечений
   - Наличие физической опоры (`is_supported`)
   - Лимит слоёв (`max_stack_layers`)
   - Правило `"лёгкие на тяжёлые"` (`allow_light_on_heavy`)

🔍 **Обоснование выбора**: FFD обеспечивает скорость и детерминированность, достаточные для логистических задач среднего масштаба. Точность упаковки на реальных наборах данных составляет `~90–95%`. Точные методы (`ILP`/`Branch & Bound`) введены как опциональные расширения (`algorithm: "ilp_ortools"`), так как для `N ≤ 50` и лимита `< 1 сек` эвристика даёт оптимальный баланс скорости/качества. В будущих версиях планируется добавление шага локального поиска (2-opt перестановки слоёв) для доводки `fill_ratio`.

## 💻 Примеры использования

### 1. Инициализация решателя
```python
from box_packer import PackingSolver

containers = [
    {'id': 'V1', 'width': 120, 'depth': 80, 'height': 100},
    {'id': 'V2', 'width': 240, 'depth': 120, 'height': 140}
]
boxes = [
    {'id': 'P1', 'width': 30, 'depth': 20, 'height': 15,
     'weight': 2.5, 'items_per_box': 10, 'max_stack_layers': None},
    {'id': 'P2', 'width': 60, 'depth': 40, 'height': 30,
     'weight': 18.0, 'items_per_box': 4, 'max_stack_layers': 3}
]

solver = PackingSolver(
    containers=containers,
    boxes=boxes,
    fill_threshold=0.9
)
```

### 2. Проверка заказа
```python
# order передаётся в количестве ИЗДЕЛИЙ (модуль сам переведёт в коробки)
order = {'P1': 200, 'P2': 40}
result = solver.check_order(order)

print(f"Заказ выполним: {result.feasible}")
print(f"Коэфф. заполнения: {result.total_fill_ratio:.2f}")
if result.unplaced_boxes:
    print(f"Не размещено: {result.unplaced_boxes}")
```

### 3. Оптимизация под порог заполнения
```python
# Найти максимальное количество изделий, при котором fill_ratio >= 0.9
max_items = {'P1': 500, 'P2': 100}
opt_result = solver.optimize(max_items)

print(f"Оптимальный заказ: {opt_result.containers_used[0].placements}")
print(f"Предупреждения: {opt_result.warnings}")
```

## ⚙️ Ограничения и правила укладки
| Правило | Описание |
|:---|:---|
| 📦 Целостность | Коробки не разрезаются. Заказ округляется до кратного `items_per_box`. |
| 🔄 Поворот | Разрешён только в горизонтальной плоскости (`0°` / `90°`). |
| 📚 Слои | Тяжёлые коробки (`max_stack_layers` задан) не могут формировать стопку выше `N` слоёв. |
| ⚖️ Вес | Лёгкие коробки (`max_stack_layers = None`) не ставятся на тяжёлые, если `allow_light_on_heavy=False`. |
| 📐 Последовательность | Контейнеры заполняются строго по порядку передачи в `containers`. |

## 📊 Производительность
- `check_order`: `< 0.8 сек` при `N ≤ 50` типов коробок и общем количестве ≤ 5000 шт.
- `optimize`: линейная сложность по количеству уменьшаемых позиций, ограничена `time_limit_seconds`.
- Память: `O(N)` для хранения слоёв и размещений.
